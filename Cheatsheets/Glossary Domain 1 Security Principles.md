# Glossary

| **Term**                           | **Definition**                                                                                                                                                          | **Source**                    |
|------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------|
| **Adequate Security**              | Security commensurate with the risk and the magnitude of harm resulting from the loss, misuse, or unauthorized access to or modification of information.                 | OMB Circular A-130            |
| **Administrative Controls**        | Controls implemented through policy and procedures. Examples include access control processes and requiring multiple personnel to conduct a specific operation.         | NIST SP 800-27 Rev A          |
| **Artificial Intelligence (AI)**  | The ability of computers and robots to simulate human intelligence and behavior.                                                                                       |                               |
| **Asset**                          | Anything of value that is owned by an organization, including both tangible (e.g., information systems) and intangible (e.g., intellectual property) assets.            |                               |
| **Authentication**                | The act of identifying or verifying the eligibility of a station, originator, or individual to access specific categories of information.                               |                               |
| **Authorization**                 | The right or permission granted to a system entity to access a system resource.                                                                                         | NIST 800-82 Rev.2             |
| **Availability**                  | Ensuring timely and reliable access to and use of information by authorized users.                                                                                      |                               |
| **Baseline**                       | A documented, lowest level of security configuration allowed by a standard or organization.                                                                            |                               |
| **Bot**                            | Malicious code that acts like a remotely controlled "robot" for an attacker, with other Trojan and worm capabilities.                                                   |                               |
| **Classified or Sensitive Information** | Information that has been determined to require protection against unauthorized disclosure and is marked to indicate its classified status.                        |                               |
| **Confidentiality**               | The characteristic of data or information when it is not made available or disclosed to unauthorized persons or processes.                                              | NIST 800-66                   |
| **Criticality**                   | A measure of the degree to which an organization depends on the information or information system for the success of a mission or business function.                   | NIST SP 800-60 Vol. 1, Rev. 1 |
| **Data Integrity**                | The property that data has not been altered in an unauthorized manner. Data integrity covers data in storage, during processing, and while in transit.                   | NIST SP 800-27 Rev A          |
| **Encryption**                    | The process of converting a message from its plaintext to ciphertext. Also referred to as enciphering.                                                                  |                               |
| **General Data Protection Regulation (GDPR)** | EU legislation passed in 2016 addressing personal privacy as an individual human right.                                                                               |                               |
| **Governance**                     | The process of how an organization is managed, including decisions, policies, roles, and procedures used for decision-making.                                           |                               |
| **Health Insurance Portability and Accountability Act (HIPAA)** | A U.S. federal law that sets standards for electronic healthcare transactions while protecting individual health information privacy.                                  | Established 1996              |
| **Impact**                         | The magnitude of harm that could be caused by a threat’s exercise of a vulnerability.                                                                                   |                               |
| **Information Security Risk**      | The potential adverse impacts to an organization’s operations, assets, individuals, other organizations, and even the nation, due to unauthorized access or disruption.  |                               |
| **Institute of Electrical and Electronics Engineers (IEEE)** | A professional organization that sets standards for telecommunications and computer engineering.                                                                  |                               |
| **Integrity**                      | The property of information ensuring its completeness, accuracy, internal consistency, and usefulness for a stated purpose.                                             |                               |
| **International Organization of Standards (ISO)** | An organization that develops voluntary international standards, particularly for information and communication technologies.                                          |                               |
| **Internet Engineering Task Force (IETF)** | The internet standards organization that defines protocol standards through collaboration and consensus.                                                           | NIST SP 1800-16B              |
| **Likelihood**                     | The probability that a potential vulnerability may be exercised within the associated threat environment.                                                              | NIST SP 800-30 Rev. 1         |
| **Likelihood of Occurrence**       | A weighted factor based on a subjective analysis of the probability that a given threat can exploit a vulnerability.                                                   |                               |
| **Multi-Factor Authentication**    | Using two or more distinct authentication factors (something you know, something you have, something you are) for identity verification.                             |                               |
| **National Institutes of Standards and Technology (NIST)** | A part of the U.S. Department of Commerce that sets standards in science and technology, including information security.                                              |                               |
| **Non-repudiation**                | The inability to deny taking an action such as creating, approving, or sending information.                                                                           |                               |
| **Personally Identifiable Information (PII)** | Information that can distinguish or trace an individual's identity, including name, Social Security number, and other personal details.                             | NIST SP 800-122               |
| **Physical Controls**              | Controls implemented through tangible mechanisms like walls, fences, guards, and locks. Many are linked to technical systems such as badge readers.                     |                               |
| **Privacy**                        | The right of an individual to control the distribution of information about themselves.                                                                               |                               |
| **Probability**                    | The chances, or likelihood, that a given threat is capable of exploiting a vulnerability.                                                                             | NIST SP 800-30 Rev. 1         |
| **Protected Health Information (PHI)** | Information related to an individual’s health status, provision of healthcare, or payment for healthcare, as defined in HIPAA.                                           |                               |
| **Qualitative Risk Analysis**      | A risk analysis method based on assigning descriptors like low, medium, or high to impact and likelihood.                                                               | NISTIR 8286                   |
| **Quantitative Risk Analysis**     | A risk analysis method where numerical values are assigned to impact and likelihood based on statistical probabilities and monetarized loss valuation.                 | NISTIR 8286                   |
| **Risk**                           | A measure of the extent to which an entity is threatened by a potential circumstance or event.                                                                        |                               |
| **Risk Acceptance**                | Deciding that the benefits of a business function outweigh the risk impact and proceeding without further action.                                                      |                               |
| **Risk Assessment**                | The process of identifying and analyzing risks to organizational operations, assets, individuals, and other organizations. Includes threat and vulnerability analysis.   |                               |
| **Risk Avoidance**                 | Deciding that the impact and/or likelihood of a specific risk is too great and not performing the related business function.                                           |                               |
| **Risk Management**                | The process of identifying, evaluating, and controlling risks, including all phases of risk context, risk assessment, risk treatment, and monitoring.                   |                               |
| **Risk Management Framework**      | A structured approach used to oversee and manage risk for an enterprise.                                                                                              | CNSSI 4009                    |
| **Risk Mitigation**                | Implementing security controls to reduce the impact and/or likelihood of a specific risk.                                                                             |                               |
| **Risk Tolerance**                 | The level of risk an entity is willing to assume to achieve a desired result.                                                                                         | NIST SP 800-32                |
| **Risk Transference**              | Paying an external party to accept the financial impact of a given risk.                                                                                             |                               |
| **Risk Treatment**                 | Determining the best way to address an identified risk.                                                                                                              |                               |
| **Security Controls**              | The management, operational, and technical controls (safeguards or countermeasures) prescribed for an information system to protect its confidentiality, integrity, and availability. | FIPS PUB 199                  |
| **Sensitivity**                    | A measure of the importance assigned to information for determining its protection needs.                                                                            | NIST SP 800-60 Vol. 1 Rev 1   |
| **Single-Factor Authentication**   | Using only one of the available authentication factors (something you know, something you have, something you are) for identity verification.                        |                               |
| **State**                           | The condition an entity is in at a point in time.                                                                                                                   |                               |
| **System Integrity**               | The quality of a system when it performs its intended function unimpaired and free from unauthorized manipulation.                                                    | NIST SP 800-27 Rev A          |
| **Technical Controls**             | Security controls primarily implemented by the information system through hardware, software, or firmware mechanisms.                                                |                               |
| **Threat**                         | Any circumstance or event with the potential to adversely impact organizational operations, assets, individuals, other organizations, or the nation through unauthorized access, destruction, modification, or denial of service. |                               |
| **Threat Actor**                   | An individual or group attempting to exploit vulnerabilities to cause or force a threat.                                                                             |                               |
| **Threat Vector**                  | The means by which a threat actor carries out their objectives.                                                                                                      |                               |
| **Token**                          | A physical object a user possesses and controls, used to authenticate the user’s identity.                                                                         | NISTIR 7711                   |
| **Vulnerability**                  | A weakness in an information system, security procedures, internal controls, or implementation that could be exploited by a threat source.                            | NIST SP 800-30 Rev 1          |
